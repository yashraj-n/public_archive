Running Hadoop in Docker: A Seamless Single-Node Setup

Author Information

Name: Yashraj Narke

Published: January 15, 2025

Reading Time: 6 minutes

Category: Big Data & DevOps

Setting up Hadoop can be a complex and time-consuming process, especially for development and testing purposes. That's why I created a Docker solution that makes running a complete Hadoop cluster as simple as executing a single command. Let me walk you through this streamlined approach to getting Hadoop up and running.

What's Included in This Setup?

My Hadoop Docker image provides a complete single-node cluster running Hadoop 3.3.6 with Java OpenJDK 11. It automatically starts all essential services:

  • HDFS Components: NameNode, DataNode, and SecondaryNameNode
  • YARN Components: ResourceManager and NodeManager
  • Web UIs: Accessible interfaces for monitoring and management

One-Command Setup

The beauty of this solution lies in its simplicity. After installing Docker, you just need to create a shared folder and run a single command. For example, on Windows:

mkdir C:\Hadoop\shared
docker run -d \
  -p 9000:9000 -p 9870:9870 -p 8088:8088 -p 8042:8042 -p 19888:19888 \
  -v C:\Hadoop\shared:/shared \
  --name hadoop-container \
  yashrajn/hadoop

Accessible Web Interfaces

Once running, you get access to several web UIs that make monitoring and managing your Hadoop cluster intuitive:

  • NameNode UI (port 9870): Monitor HDFS health and browse the filesystem
  • YARN ResourceManager UI (port 8088): Track running applications and cluster resources
  • NodeManager UI (port 8042): View individual node statistics
  • JobHistory UI (port 19888): Analyze completed MapReduce jobs

Working with HDFS

The container comes pre-configured with HDFS accessible at hdfs://localhost:9000. The shared folder mounted at /shared makes it easy to transfer files between your host system and HDFS. You can quickly upload files and start working with distributed storage without complex configuration.

Why This Matters

This Docker approach eliminates the traditional pain points of Hadoop setup - no more wrestling with Java versions, configuration files, or service dependencies. Whether you're learning Hadoop, developing applications, or need a quick testing environment, this solution gets you productive in minutes rather than hours.

The project is open source and available on GitHub, with the Docker image hosted on Docker Hub for easy access. It's designed to be a zero-hassle solution for anyone who needs Hadoop running quickly and reliably.
